# Savva Stealth System (SSS) ⚡

**Savva Stealth System** — консольная утилита для повышения анонимности и сетевой безопасности в Linux. Скрипт автоматизирует комплексную настройку системы: от рандомизации физических идентификаторов до создания защищенного сетевого контура.

> **Статус проекта:** `Alpha / Experimental` ⚠️  
> **Тестировалось на:** `Arch Linux`

---

## ⚠️ ОТКАЗ ОТ ОТВЕТСТВЕННОСТИ
Использование этого ПО может привести к потере сетевого доступа, нарушению работы сервисов и потере данных. 
* Запускайте **только в тестовой среде** (VM, Live USB).
* Используйте на свой страх и риск. 
* Всегда делайте бэкапы и проверяйте код перед запуском.

---

## 🛠 Что делает скрипт сейчас

*   **Идентификация:** Генерация случайного `hostname` и MAC-адреса на уровне L2 (включая режим сканирования).
*   **DNS Protection:** Настройка локального резолвера `Unbound` с **DNS-over-TLS**. Блокировка подмены DNS через DHCP.
*   **Privacy:** Жёсткое отключение логирования запросов, минимизация QNAME и скрытие версий служб.
*   **Network Hardening:** Полное отключение IPv6 на уровне ядра для предотвращения утечек.
*   **Kill-Switch:** Настройка `iptables` для блокировки всего трафика вне белого списка (Tor, DoT, HTTP/S).
*   **Anti-Forensics:** Ручное управление системным временем и отключение NTP-синхронизации.
*   **Audit:** Реалтайм-мониторинг активных соединений и состояния защитных механизмов.

## 🛡 От чего SSS защищает твою систему

### Сетевая идентификация (L2/L3)
*   **Трекинг по MAC:** Мешает отслеживать перемещения между Wi-Fi точками.
*   **Деанонимизация по Hostname:** Скрывает личность в логах маршрутизаторов.
*   **DHCP-манипуляции:** Защищает от навязанных шпионских DNS-серверов.

### Приватность трафика (DNS/IPv6)
*   **Перехват DNS (MITM):** Шифрование скрывает посещаемые сайты от провайдера.
*   **Утечки через IPv6:** Блокирует «черные ходы» трафика, через которые виден реальный IP.
*   **Локальное слежение:** Нулевое логирование гарантирует отсутствие истории запросов на диске.

### Контроль и отпечаток
*   **Kill-Switch:** Запрещает ПО выходить в сеть напрямую мимо защиты.
*   **Fingerprinting:** Мешает вычислить местоположение по часовым поясам и версии ОС.

## 📦 Установка и запуск

Скрипт содержит встроенный инсталлятор, который автоматически подтягивает зависимости (`unbound`, `tor`, `curl` и др.) через `pacman` или `apt`.

### Запуск
```bash
# Рекомендуется сначала запустить в режиме эмуляции:
sudo python main.py --dry-run

# Выполнение изменений:
sudo python main.py --confirm
```
# Рекомендации по бэкапам (сохранять до запуска)

    Сделайте резервные копии ключевых конфигураций: sudo cp /etc/resolv.conf /etc/resolv.conf.bak
    sudo cp -r /etc/unbound /etc/unbound.bak
    sudo cp /etc/NetworkManager/NetworkManager.conf /etc/NetworkManager/NetworkManager.conf.bak
    sudo cp /etc/hosts /etc/hosts.bak
    sudo cp /etc/hostname /etc/hostname.bak
    Для сохранения текущего состояния iptables: sudo iptables-save > ~/iptables.rules.bak
    Для сохранения sysctl и настроек IPv6: sudo cp /etc/sysctl.conf /etc/sysctl.conf.bak
    Если изменяете системное время — зафиксируйте текущее аппаратное время:** sudo hwclock --systohc

# Лучшие практики перед запуском

    Запускайте только в изолированной среде (VM, Live USB) при первом тестировании.
    Убедитесь, что у вас есть физический или консольный доступ для восстановления, если потеряете сетевой доступ.
    Не храните в репозитории реальные ключи, пароли или мосты Tor — используйте шаблоны/placeholder'ы.
    Проверяйте историю git на возможные утечки секретов и очищайте при необходимости.
